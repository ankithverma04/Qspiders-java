# day 12
### Work Summary
On the twelfth day of the internship, I learned about non-primitive variables in Java, also known as reference variables. These variables store the memory address of objects instead of storing actual values like primitive variables. The session explained how objects are created in memory and how reference variables point to those objects. I also learned how object references can be compared using the equality operator (==). Through examples, I understood that the equality operator compares whether two references point to the same object rather than comparing their internal data. This topic helped me understand how Java manages objects in memory.

### Learning Outcomes

Through this session, I gained a clear understanding of reference variables and how they differ from primitive variables in Java. I learned that reference variables store the memory address of objects rather than the actual data. I also understood how multiple reference variables can point to the same object in memory. Practicing object comparison using the equality operator helped me understand how Java checks whether two references refer to the same object. This knowledge improved my understanding of memory management and object handling in Java and strengthened my foundation in object-oriented programming concepts.
